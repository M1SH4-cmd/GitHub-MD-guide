
# Инструкция по языку Markdown

## Что такое Markdown

**Markdown** — это простой язык разметки текста, который позволяет удобно оформлять документы без сложных редакторов.  
С его помощью можно создавать:

- заголовки;
- списки;
- ссылки;
- изображения;
- таблицы;
- блоки кода;
- цитаты;
- выделение текста.

Markdown часто используется на:

- GitHub;
- GitLab;
- в документации проектов;
- в `README.md`;
- в заметках и учебных материалах.

---

## Зачем нужен Markdown

Markdown удобен тем, что:

- его легко выучить;
- текст быстро оформляется;
- файлы остаются простыми и читаемыми;
- он отлично подходит для документации и инструкций;
- его поддерживает GitHub.

## Заголовки

Заголовки создаются с помощью символа `#`.

```markdown
# Заголовок 1 уровня
## Заголовок 2 уровня
### Заголовок 3 уровня
#### Заголовок 4 уровня
##### Заголовок 5 уровня
###### Заголовок 6 уровня
````

### Результат

# Заголовок 1 уровня

## Заголовок 2 уровня

### Заголовок 3 уровня

#### Заголовок 4 уровня

##### Заголовок 5 уровня

###### Заголовок 6 уровня

---

## Абзацы и перенос строки

Обычный текст пишется просто строками.

```markdown
Это первый абзац.

Это второй абзац.
```

Если нужен перенос строки внутри абзаца, можно оставить два пробела в конце строки или начать новую строку.

```markdown
Первая строка.  
Вторая строка.
```

### Результат

Первая строка.
Вторая строка.

## Выделение текста

### Курсив

```markdown
*Курсив*
_Курсив_
```

*Курсив*
*Курсив*

### Полужирный текст

```markdown
**Полужирный**
__Полужирный__
```

**Полужирный**
**Полужирный**

### Полужирный курсив

```markdown
***Полужирный курсив***
```

***Полужирный курсив***

### Зачеркнутый текст

```markdown
~~Зачеркнутый текст~~
```

~~Зачеркнутый текст~~

## Списки

### Маркированный список

```markdown
- Первый пункт
- Второй пункт
- Третий пункт
```

* Первый пункт
* Второй пункт
* Третий пункт

Можно использовать и символ `*`:

```markdown
* Первый пункт
* Второй пункт
* Третий пункт
```

### Нумерованный список

```markdown
1. Первый пункт
2. Второй пункт
3. Третий пункт
```

1. Первый пункт
2. Второй пункт
3. Третий пункт

### Вложенные списки

```markdown
1. Первый пункт
   - Подпункт
   - Подпункт
2. Второй пункт
```

1. Первый пункт

   * Подпункт
   * Подпункт
2. Второй пункт

## Ссылки

Ссылка оформляется так:

```markdown
[Текст ссылки](https://example.com)
```

Пример:

```markdown
[Открыть GitHub](https://github.com)
```

Результат:
[Открыть GitHub](https://github.com)

---

## Изображения

Изображение оформляется почти так же, как ссылка, но с `!` в начале.

```markdown
![Описание изображения](https://example.com/image.png)
```

Пример:

```markdown
![Логотип GitHub](https://github.githubassets.com/images/modules/logos_page/GitHub-Mark.png)
```

## Цитаты

Для цитаты используется символ `>`.

```markdown
> Это цитата.
```

> Это цитата.

Можно делать многострочную цитату:

```markdown
> Это первая строка цитаты.
> Это вторая строка цитаты.
```

> Это первая строка цитаты.
> Это вторая строка цитаты.

## Код

### Встроенный код

Если нужно выделить одно слово или команду внутри текста, используются одинарные обратные кавычки.

```markdown
Команда `git status` показывает состояние репозитория.
```

Результат:
Команда `git status` показывает состояние репозитория.

## Блоки кода

Для блока кода используются три обратные кавычки.

<pre lang="markdown"><code>```markdown
Тут может быть текст или код
в нескольких строках
```</code></pre>

Пример:

<pre lang="markdown"><code>```bash
git add .
git commit -m "Обновлен README"
git push
```</code></pre>

### Результат

```bash
git add .
git commit -m "Обновлен README"
git push
```

---

## Указание языка в блоке кода

После первых трёх кавычек можно указать язык. Это нужно для подсветки синтаксиса.

Примеры:

<pre lang="markdown"><code>```python
print("Hello, world!")
```</code></pre>

<pre lang="markdown"><code>```cpp
#include &lt;iostream&gt;
using namespace std;

int main() {
    cout &lt;&lt; "Hello, world!";
    return 0;
}
```</code></pre>

<pre lang="markdown"><code>```html
&lt;h1&gt;Привет&lt;/h1&gt;
```</code></pre>

## Таблицы

Таблицы в Markdown создаются с помощью вертикальных черт `|` и дефисов `-`.

```markdown
| Имя | Группа | Оценка |
|-----|--------|--------|
| Иван | ИС-21 | 5 |
| Анна | ИС-22 | 4 |
```

### Результат

| Имя  | Группа | Оценка |
| ---- | ------ | ------ |
| Иван | ИС-21  | 5      |
| Анна | ИС-22  | 4      |

### Выравнивание в таблицах

```markdown
| Лево | Центр | Право |
|:-----|:-----:|------:|
| Текст | Текст | Текст |
```

### Результат

| Лево  | Центр | Право |
| :---- | :---: | ----: |
| Текст | Текст | Текст |

```markdown

```

### Результат

## Экранирование символов

Если нужно показать специальные символы как обычный текст, используется обратный слеш `\`.

Пример:

```markdown
\*Это не курсив\*
```

Результат:
*Это не курсив*

- `git push` — отправить изменения

* `git add .` — добавить изменения

## Полезные советы по Markdown

1. Не делайте слишком длинные абзацы.
2. Используйте заголовки для структуры текста.
3. Для команд всегда применяйте блоки кода или встроенный код.
4. Для инструкций удобно использовать списки.
5. Для описания данных хорошо подходят таблицы.
6. Проверяйте, как файл отображается на GitHub.

## Частые ошибки

### Ошибка 1. Нет пробела после символа `#`

Неправильно:

```markdown
#Заголовок
```

Правильно:

```markdown
# Заголовок
```

### Ошибка 2. Неправильно оформлен список

Нужно ставить пробел после `-`.

Неправильно:

```markdown
-пункт
```

Правильно:

```markdown
- пункт
```

### Ошибка 3. Не закрыт блок кода

Если открыть блок тремя кавычками, его нужно обязательно закрыть.

Неправильно:

<pre lang="markdown"><code>```bash
git status</code></pre>

Правильно:

<pre lang="markdown"><code>```bash
git status
```</code></pre>
